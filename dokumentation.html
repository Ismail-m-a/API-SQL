Dokumentation av API

Routes

  • GET localhost:3000/members - returnerar alla användare.

  • GET localhost:3000/members/{id} - returnerar en användare med angivet id eller status 404 om användaren saknas.

  • POST localhost:3000/addMembers - skapar en ny user med hashat lösenord, accepterar JSON objekt på formatet {"username": "", "password": "", "name": "", "email": ""}.
    Alla användarobjektet är obligatoriska och ska vara korrekt fyllda. Username ska vara unik och får inte vara i användning.

  • PUT localhost:3000/members/:id - uppdaterar user med angivet id, dvs user som redan finns i databasen med möjlighet att välja ett eller flera användarobjekt.
    Accepterar JSON-objekt på formatet {"username": "", name": "", "password": "", "email": ""}.

  • POST localhost:3000/login - inloggning, accepterar JSON-objekt på formatet {"username": "", "password": ""}.
    Returnerar inte information om användaren i klartext utan en tidsbegränsad JWT (token).

  • DELETE localhost:3000/deleteMembers/{id} - raderar en user med angiven id. exempel: ( localhost:3000/deleteMembers/25 )
